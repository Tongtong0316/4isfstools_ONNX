## 需要特别注意的点

0. 平台分治
- macOS 当前可用，不要因为 Windows 调试顺手改坏 macOS
- Windows 修复优先做平台分支或运行时隔离
- 修改共享文件时要说明影响平台

1. 取消逻辑
- 取消必须作用到整个进程组
- 新任务启动前要清理旧 cancel 标记和 job 状态
- 取消后的状态不能被后台收尾逻辑覆盖回 ready/error

2. 歌词质量
- 当前仍然是“候选歌词 + 人工校正”路线
- 如果要继续提升质量，优先考虑：
  - 更好的歌词文本来源
  - 更强的文本切分
  - 更细粒度对齐

3. 播放稳定性
- 切歌、切模式、删除当前歌曲时要先清理旧音频引用
- `AbortError` / `MEDIA_ERR_ABORTED` 仍应视作可预期中断，不应误报为失败

## 建议下一步
