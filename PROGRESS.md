## 已落地（可运行）

1. 全局拖拽导入
- 根容器接管拖拽事件，支持直接导入音频文件

2. 歌词文档体系
- 后端新增 `LyricDocument` 结构
- 处理完成后同时生成 `lyrics.json` 与 `lyrics.lrc`
- 新增命令：
  - `get_lyrics_document(song_id)`
  - `save_lyrics_document(song_id, document)`

3. 歌词编辑交互
- 可在流式歌词中直接编辑文本
- 编辑后去抖自动保存（400ms）
- 支持行级 ±100ms 微调
- 点击歌词行可跳播

4. UI 布局调整（已固化）
- 顶部按钮与状态区间距优化
- 底部播放器与窗口底边留出明确安全距离

5. 播放中断错误处理修复
- 将 `AbortError / MEDIA_ERR_ABORTED` 识别为可预期中断（切歌/切模式时）
- 仅对真实播放失败显示红色错误文案
- 避免“实际可继续播放但 UI 显示失败”的误导

6. 歌词滚动居中体验
- 播放时自动把当前唱行保持在中线附近
- 用户滚动歌词时临时退出自动回中，避免“抢滚动焦点”
- 用户停止滚动约 0.9 秒后自动恢复回中
- 暂停或未播放状态不强制回中，允许停在任意位置
- 列表顶部与底部加入动态留白，首尾歌词可在视觉上自然居中
- 居中算法由 `scrollIntoView` 升级为容器 `scrollTop` 精确计算，修复播放中偶发不回中的问题
- 程序触发的平滑滚动保护时长延长，避免被误判为用户滚动后锁死回中
- 歌词文本层改为几何中心布局，解决“看起来没回中（水平偏移）”的问题

7. 播放列表组织
- 播放列表支持文件夹分组、折叠/展开和拖拽移动
- 右键菜单新增重命名、移动到、重新生成歌词等操作
- 文件夹状态保存在本地，便于下次打开时恢复

8. 歌词引擎升级
- 现已移除本地转写/听写管线，歌词能力保留为候选搜索、手动编辑与保存

9. 取消处理可靠性
- 取消时按进程组终止 Demucs / Whisper 子进程，避免只杀父进程导致后台继续执行
- 新一轮处理开始前会清理残留取消标记，避免取消状态污染后续任务
- 交接入口已固化到 `HANDOFF.md`，便于下一位 agent 直接接手

10. 文件夹创建体验
- 播放列表“新建文件夹”改为显式对话框输入，确保点击后一定可见可交互
- 对话框已进一步增宽并增加 footer 间距，减少视觉拥挤
- 底部按钮改成更饱满的胶囊，避免文字贴边

11. 播放模式切换行为
- 选歌或重新开始播放时仍从 0 起播
- 切换 `原唱` / `伴奏` / `人声` 时保留当前进度，仅改变轨道音量与路由

12. Windows 便携交付（进行中）
- 新增 Windows 便携构建脚本：`scripts/build-windows-portable.ps1`
- 新增 npm 命令：`npm run win:portable`
- 便携交付目标：ZIP 解压后直接运行 `forisfstools.exe`
- 一键补齐链路新增：
  - Windows 下 `FFmpeg` 缺失时尝试 `winget` 自动安装
  - Windows 下 Python 运行时缺失时尝试 `winget` 自动安装并写入运行时路径提示
  - 核心模块 (`torch` / `demucs`) 优先离线拷贝，失败后再尝试在线安装（中国大陆可达源）
  - 模型安装新增 `runtime-manifest.json` 入口：优先本地离线模型，缺失时按 manifest 中大陆可达源尝试下载并解压到 runtime/models
  - manifest 已升级为分平台结构（macOS / Windows）并支持 `sha256` 校验字段，减少“多 runtime 来源”误判
  - 安装结果提示修正：前端根据 `canRunCore` 判断，不再出现“安装完成”与缺件并存的矛盾文案
  - 后端低风险拆分第一步：新增 `process_control.rs` 承担平台进程控制（Unix 进程组 / Windows taskkill），降低后续 Windows 便携交付风险
